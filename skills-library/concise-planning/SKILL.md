# Concise Planning Skill

## Propósito
Garantizar que el agente siempre genere un plan técnico estructurado antes de escribir código. Esto previene implementaciones fragmentadas o desorganizadas y proporciona una hoja de ruta clara.

## Instrucciones
1. **Analizar la solicitud:** Evalúa los requisitos del usuario para la tarea o característica solicitada.
2. **Generar un plan:** Crea una lista de tareas (checklist) atómica, clara y accionable.
3. **Pausar ejecución:** No empieces a generar o modificar código fuente inmediatamente.
4. **Desglosar tareas:** Divide las tareas complejas en pasos lógicos, modulares y secuenciales. Presenta este plan al usuario para su validación.

## Mejores Prácticas y Restricciones
- **Mantenlo conciso:** Evita explicaciones teóricas largas; enfócate directamente en la lista de tareas.
- **Accionabilidad:** Cada paso debe representar una acción concreta del desarrollo (ej. "Crear componente de UI", "Añadir ruta en el backend").
- **Ámbito:** Sigue el principio de una sola responsabilidad por tarea.
