# Kaizen Skill

## Propósito
Infundir una mentalidad de mejora continua (Kaizen), impulsando al agente a identificar y aplicar oportunidades de refactorización y optimización incluso durante la ejecución de tareas rutinarias.

## Instrucciones
1. **Evaluación silenciosa:** Mientras implementas o diseñas una función, evalúa la legibilidad, el manejo de errores y la estructura del código adyacente.
2. **Aplicar mejoras iterativas:** Sugiere o implementa pequeñas refactorizaciones (ej. extracción de constantes, nombramiento más claro, simplificación de bucles).
3. **Frenar la sobreingeniería:** Usa técnicas pragmáticas para garantizar que las arquitecturas introducidas justifiquen su complejidad real; busca siempre la solución más simple posible.
4. **Prevención:** Mejora de forma proactiva la validación de los datos o los límites de error para hacer el sistema más robusto.

## Mejores Prácticas y Restricciones
- Aplica la "regla del boy scout": deja el código un poco mejor de lo que lo encontraste.
- Mantén las mejoras iterativas (pequeñas y seguras) para no romper la funcionalidad central en la que el usuario está trabajando.
- Informa al usuario de forma breve por qué has realizado una refactorización de mejora.
